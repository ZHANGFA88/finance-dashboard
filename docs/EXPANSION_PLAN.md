# FinSight 扩展整合方案（Day3+）

> 目的：把「Mstock 异动监控」+「筹码分析技能」+「盘中分析技能」整合进 FinSight，
> 让它从"看行情大屏"升级为"个人盯盘工作站"。
> 本文只是方案，未开工。经确认后按里程碑分步执行，每步可验收。

---

## 一、现状（已完成）

- **Day1 后端**：serve.py（纯标准库）+ SQLite + 多源容灾（新浪/东财/Yahoo）+ 3 API + 13 只自选股
- **Day2 前端**：科幻大屏 S1–S7（骨架/状态栏/行情卡片/15s轮询/3D地球/K线弹窗/涨跌动画）
- 服务：`0.0.0.0:8770`，静态资源禁缓存，git 已存档（437047f）

FinSight 现在能力 = **看**（行情一目了然）。缺 **盯**（报警）和 **问**（分析）。

---

## 二、三个待整合模块

| 模块 | 来源 | 形态 | 补的能力 |
|---|---|---|---|
| ① 异动监控报警 | Mstock | 独立程序（2924行） | 盯 —— 超阈值自动叫你 |
| ② 筹码分布分析 | stock-cyq-analyzer | 脚本(akshare) | 问 —— 筹码/成本/获利盘 |
| ③ 盘中技术分析 | a-share-intraday | 方法论(SKILL.md) | 问 —— 技术面+多空+操作建议 |

**取舍原则**：只搬对"内网个人大屏"有用的，跳过 Mstock 的整套安全模块（登录/CSRF/限流/多用户——公网多用户才需要，我们用不上）。

---

## 三、目标架构

```
                    ┌───────────────────────────┐
                    │      FinSight 大屏 (前端)     │
                    │  行情卡片 │ 3D地球 │ K线弹窗   │
                    │  + 异动提示条  + 分析面板       │
                    └───────────┬───────────────┘
                                │ HTTP API
                    ┌───────────┴───────────────┐
                    │      serve.py (主后端)       │
                    │  /quotes /kline /watchlist   │
                    │  + /alerts (异动)            │
                    │  + /analyze (盘中分析)        │
                    └──┬─────────┬────────┬───────┘
                       │         │        │
              后台盯盘线程   指标计算    调用独立环境
              (双窗+冷却)  (MA/MACD..)  (筹码 akshare)
                       │                  │
                   SQLite alerts表     /tmp/cyqenv (隔离)
```

**关键决策**：
- ①③ 用纯标准库 + 现有数据源，**直接进主后端**（轻）
- ② 筹码依赖 akshare（重），**独立虚拟环境**，主后端用 subprocess 调用，不污染主服务

## 四、里程碑拆解（每步可独立验收）

### 🅐 里程碑 A：异动监控报警（对应 Mstock）

| 步 | 内容 | 验收标准 |
|---|---|---|
| A1 | 建 `alerts` + `monitor_config` 表；抠 Mstock 双窗算法（长窗/短窗）+ 冷却机制 | 单元测试：喂模拟数据能正确判定异动 |
| A2 | 后台盯盘线程：轮询自选股，算涨跌速度，超阈值写 alert | 日志能看到"XX触发异动" |
| A3 | 加腾讯行情源（`stock.gtimg.cn`）进多源容灾 | Yahoo挂时腾讯能顶上 |
| A4 | `/api/finance/alerts` API + 大屏顶部**异动提示条**（滚动/闪烁） | 触发时大屏能看到提示 |
| A5 | （可选）飞书/TG 推送 | 手机能收到报警 |

**估时**：1~1.5 天　**难点**：A股交易时段判断、盘中真实触发验证

### 🅑 里程碑 B：盘中技术分析（对应 a-share-intraday）

| 步 | 内容 | 验收标准 |
|---|---|---|
| B1 | 实现指标计算：MA5/10/20/60、MACD、KDJ、RSI14（纯 Python） | 对比东财数据，指标值一致 |
| B2 | 补大盘指数 + 板块热点数据源 | API 能返回上证/深成/板块涨幅 |
| B3 | 多空分歧判断框架（按 SKILL.md 的量价/影线/封单逻辑） | 输出结构化判断 |
| B4 | `/api/finance/analyze?symbol=` + K线弹窗加"深度分析"面板 | 点个股能看到技术面解读 |
| B5 | （可选）接 AI 生成自然语言解读 | 一段话讲清这只股 |

**估时**：1.5~2 天　**难点**：指标算法正确性、AI 编排

### 🅒 里程碑 C：筹码分布分析（对应 stock-cyq-analyzer）

| 步 | 内容 | 验收标准 |
|---|---|---|
| C1 | 建独立环境 `/tmp/cyqenv` 装 akshare/numpy/pandas/matplotlib | 能跑通示例 `cyq_report.py 600206` |
| C2 | 主后端 `/api/finance/cyq?symbol=` 用 subprocess 调用，返回指标+图 | 返回获利盘/成本区/集中度 |
| C3 | K线弹窗加"筹码"标签页，显示筹码图 + 指标 | 点开能看到筹码分布图 |

**估时**：1 天　**难点**：akshare 装依赖慢/易踩坑，环境隔离

## 六、风险与对策

| 风险 | 对策 |
|---|---|
| akshare 依赖重、拖慢主服务 | 独立虚拟环境 + subprocess 隔离，绝不进主后端 |
| 数据源限流(Yahoo 429) | 多源容灾 + 加腾讯源 + curl 兜底(已做) |
| 战线太长中途卡住 | 里程碑制，每个 A/B/C 独立可交付，先做完 A 再动 B |
| 指标算错 | 每个指标对比东财/新浪真实值校验 |
| 交易时段外无数据 | 加 A股交易日/时段判断（借 Mstock 逻辑） |

---

## 七、建议

1. **不全搬**：筹码(②)依赖重、盘中AI(③)工程大，一次上三个战线太长。
2. **先做 A（异动报警）**：收益最直接、纯标准库最稳、1 天能交付验收。
3. **②③ 可先做成独立技能**：不一定嵌大屏，让 AI 平时能调用分析，成熟后再嵌。
